01. El Desafío: Entender al Cliente

En el comercio electrónico, no todos los clientes tienen el mismo valor. Este proyecto aborda una pregunta fundamental de negocio: ¿Cómo identificar qué clientes generan más valor y cuáles están en riesgo de abandono?

Desarrollé un sistema completo de segmentación RFM (Recency, Frequency, Monetary) que transforma datos transaccionales brutos en segmentos de clientes accionables para estrategias de marketing y retención.

02. Pipeline de Datos

El proceso completo se ejecuta en dos fases principales usando Python/Pandas y SQLite para procesamiento híbrido:

FASE_1: LIMPIEZA

Filtrado de transacciones inválidas, manejo de nulos, cálculo de ingresos totales y preparación de fechas.

FASE_2: CÁLCULO RFM

Cálculo de Recencia (Python), Frecuencia y Monetario (SQL), scoring y segmentación de negocio.

# Carga y Limpieza
df = pd.read_csv('online_retail.csv', encoding='ISO-8859-1')
df['Sales_Total'] = df['Quantity'] * df['UnitPrice']
df_cleaned = df[(df['UnitPrice'] > 0) & (df['Quantity'] > 0)]
df_cleaned.dropna(subset=['CustomerID'], inplace=True)

# Cálculo de Recencia
snapshot_date = df_cleaned['InvoiceDate_dt'].max() + pd.Timedelta(days=1)
df_recency = df_cleaned.groupby('CustomerID')['InvoiceDate_dt'].max().reset_index()
df_recency['Recency'] = (snapshot_date - df_recency['Last_Purchase_Date']).dt.days

# Frecuencia y Monetario (SQL)
sql_query = """
SELECT CustomerID,
COUNT(DISTINCT InvoiceNo) AS Frequency,
ROUND(SUM(Sales_Total), 2) AS Monetary
FROM RetailCleaned
GROUP BY CustomerID
"""

04c. Key Learnings

  • El valor no es lineal: Un pequeño porcentaje de clientes (2.7% "No se pueden perder") puede representar una porción desproporcionada del revenue histórico.
  • La recencia es el predictor más fuerte: Un cliente que no compra hace 90+ días tiene probabilidad drásticamente menor de volver, independientemente de su valor histórico.
  • Segmentación accionable > Segmentación perfecta: 7 segmentos bien definidos son más útiles que 20 clusters estadísticamente perfectos pero incomprensibles para marketing.
  • Python + SQL es un combo poderoso: Python para limpieza y recencia, SQL para agregaciones masivas de frecuencia y monetario.
